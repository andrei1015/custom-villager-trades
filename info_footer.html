Customisation&Development
This idea is a combination and adaptation of two ideas. Well, one was the fact that I wanted custom player heads on my server and not being able to use an anvil to rename a player head for it to get the skin is f****** annoying MOJANG. The first idea was Xisuma's video on custom player heads on the Hermitcraft server. The second idea was MadCat's datapack that comes with a ruby program that generates the file you need for the datapack to work. That datapack implements player head drop when killing someone. I did not want to encourage that on my server so I just combined the ideas. Awesome stuff, custom heads for everyone!

Also thanks to this dude right here.
Limitations
There are a few limitations. The first one is the execution time for the script, which is curretnly at 2:30 minutes. This means the more different player heads you feed it to generate, the more time it takes, and if it goes above the execution time limit, bad stuff happens. The second one is 1 skin fetch per player per minute. That's spam protection from Mojang on their API.
